# 스테이 압해 구축 로드맵

> 2026-07-19 기준. 완료된 것과 앞으로 갈 길을 단계별로 기록한다.
> 각 단계는 "게스트가 체감하는 가치"가 생기는 단위로 자른다 — 기술을 위한 기술 단계는 없다.

## 현재 완료 상태 (v1)

- 브랜드 디자인 시스템 (크림/잉크/스톤/브론즈, Noto Serif KR, 리빌 애니메이션)
- "스테이 압해" 리브랜딩 + 단일 독채(하루 한 팀) 콘셉트 정립
- 공개 예약 폼: 가용성 달력(확정 예약+외부 캘린더 차단일) → pending 접수 → 운영자 알림(텔레그램/구글챗)
- 관리자: 로그인, 콘텐츠 관리(랜딩 연동), 예약 승인/취소, 외부 캘린더 동기화
- DB 레벨 이중예약 방지(EXCLUDE 제약), iCal 양방향 동기화(수신 크론 + 송신 피드)

## Phase 1 — 예약 운영 완성 (4순위 + α)

**목표: 게스트가 "접수됐는지, 확정됐는지" 사이트 바깥에서 확인 가능. 선행: 0-2 완료.**

- [ ] **접수 확인 메일** (Resend): 예약 접수 직후 게스트에게 자동 발송. 기존 `src/lib/notifications/` 채널 패턴에 `resend.ts` 추가하는 구조
- [ ] **확정/취소 메일**: 관리자가 승인/취소하는 서버 액션에 훅. 확정 메일에 주소·체크인 안내·(향후) 컨시어지 QR 링크 포함
- [ ] **체크인 D-1 리마인더**: 크론 하나 추가 (내일 체크인 예약 조회 → 발송)
- [ ] 후속(선택): 카카오 알림톡 전환 — 국내 게스트 도달률은 메일보다 압도적. 단 사업자·발신프로필 심사 필요하니 메일로 시작 후 전환
- [ ] 예약 조회 페이지(선택): 접수 시 발급한 코드로 게스트가 상태 확인 — 문의 전화를 줄여줌

## Phase 2 — 결제·수익화

**목표: "입금 확인 후 수동 확정"을 자동화하고 노쇼를 줄인다.**

- [ ] 요금 모델 설계: `pricing_rules` 테이블(기본가·주말가·성수기 기간별 가격) → 예약 폼에서 총액 자동 계산·표시 (`total_price` 컬럼은 이미 존재)
- [ ] 토스페이먼츠 연동: 예약금(또는 전액) 결제 → 결제 완료 시 자동 `confirmed` 전환. EXCLUDE 제약이 동시 결제 경합의 최종 방어선이 되는 구조 그대로 활용
- [ ] 취소/환불 정책 페이지 + 환불 규정 자동 계산(체크인까지 남은 일수 기준)
- [ ] 현금/계좌이체 병행 옵션 유지 (초기엔 수동 확정 경로도 남겨둔다)

## Phase 3 — 압해 컨시어지 (AI 스테이 프로그램)

**목표: 랜딩에서 약속한 9개 기능을 실제로 제공. 정적 콘텐츠 → 인증 → AI 순으로 점진 구축.**

- [x] **3a. 게스트 가이드 페이지** (`/guide`): 이용 매뉴얼·오시는 길(지도앱 딥링크)·주인장 로컬 맛집·추천 코스(천사대교/퍼플섬/목포 원도심)·시크릿 쿠폰 안내. **비공개 정책 적용** — 공개 링크 없음·noindex·sitemap 제외, 예약 확정 고객에게 QR/링크로만 공유(랜딩은 소개만). ⚠️ 맛집 상호·충전소 위치는 플레이스홀더 — 실제 정보로 교체 필요
- [ ] **3b. 투숙객 인증**: 확정 메일/QR에 담긴 예약 코드로 투숙 기간에만 열리는 전용 영역 (바베큐 예약 신청, T맵 목적지 전송 링크, 시크릿 쿠폰)
- [ ] **3c. AI 챗 컨시어지**: Claude API 기반 대화형 안내 — 가이드 콘텐츠+예약 정보를 컨텍스트로, "지금 갈만한 노을 스팟" 같은 질문 응대. `concierge_logs` 테이블(스키마 구상 완료)에 요청 기록
- [ ] **3d. IoT 컨트롤**: 조명·냉난방 연동 (장비 선정 필요, 가장 후순위)

## Phase 4 — 데이터·마케팅 자동화

**목표: 감(感)이 아니라 데이터로 공실을 줄인다.**

- [ ] GA4 설치 + 핵심 이벤트(`reservation_submit`, `click_youtube_aphaedo` 등) → 유입 채널별 예약 전환율 파악
- [ ] **후기 수집 루프**: 체크아웃 다음날 자동 메일 "하루는 어떠셨나요?" → 회신을 관리자가 `/admin/content` testimonial 블록으로 승인 게시 (이미 만든 후기 섹션이 자동으로 채워지는 구조)
- [ ] 공실 관리: 2주 내 빈 날짜 감지 → 인스타 스토리용 문구 생성 / 재방문 고객 할인 링크 발송 (architecture-rules의 "능동적 공실 관리" 구상 실현)
- [ ] 월간 리포트: 예약률·채널별 유입·매출 요약을 운영자에게 자동 발송
- [ ] (규모 커지면) BigQuery + Looker Studio — 초기엔 GA4 기본 리포트로 충분
